# Платформер

Двухмерный платформер на Phaser 3 (без сборки). Три уровня, монеты, враги, движущиеся платформы.

---

## Запуск

Игра использует ES-модули, поэтому открыть `index.html` напрямую через `file://` **не получится** — нужен локальный HTTP-сервер.

```bash
# Вариант 1 — Python (есть почти везде)
python3 -m http.server 8000

# Вариант 2 — Node.js
npx serve .
```

Затем открыть в браузере: **http://localhost:8000**

## Тесты

npm test

- `test/parseLevel.test.js` — парсер уровней
- `test/progress.test.js` — логика жизней и прогресса

## Управление

| Экран | Клавиши |
|---|---|
| Меню | Клик мышью по уровню **или** `Enter` — начать с первого уровня |
| Игра — движение | `←` / `→` или `A` / `D` |
| Игра — прыжок | `↑` / `W` / `Пробел` (только стоя на поверхности) |
| Уровень пройден | `Пробел` — следующий уровень, `M` — в меню |
| Игра окончена | `R` — заново тот же уровень, `M` — в меню |

## Игровые механики

- **3 жизни** на уровень. Жизнь теряется при: касании шипа, падении в яму, контакте с врагом сбоку.
- **Прыжок на врага сверху** — убивает врага, игрок отскакивает вверх.
- **Монеты** дают очки (по 10). Счёт накапливается между уровнями.
- **Движущиеся платформы** везут игрока, когда он стоит сверху (передвижение по X и Y).
- **Флаг** в конце уровня — победа. Жизни сбрасываются до 3 при переходе, счёт сохраняется.
- После прохождения всех 3 уровней — экран **«ИГРА ПРОЙДЕНА!»**.

## Структура проекта

```
index.html                  — точка входа; подключает Phaser 3.80.1 с CDN
package.json                — только скрипт npm test (зависимостей нет)
src/
  main.js                   — создаёт экземпляр Phaser.Game со всеми сценами
  config.js                 — CONFIG (игровые константы) и COLORS (palette placeholder)
  assets.js                 — ASSET_KEYS (реестр ключей/размеров) + createPlaceholderTextures()
  levels.js                 — массив LEVELS: 3 уровня (ASCII-карта, враги, платформы)
  scenes/
    BootScene.js            — генерация placeholder-текстур → запуск Menu
    MenuScene.js            — список уровней, выбор мышью или Enter
    GameScene.js            — основная игровая сцена (мир, физика, коллизии)
    UIScene.js              — HUD (жизни, счёт, номер уровня), работает поверх GameScene
    GameOverScene.js        — экран «ИГРА ОКОНЧЕНА», R / M
    LevelCompleteScene.js   — экран «УРОВЕНЬ ПРОЙДЕН!», Пробел / M
  entities/
    Player.js               — спрайт игрока, управление, прыжок
    Enemy.js                — враг с патрулированием, разворот на краях
    MovingPlatform.js       — движущаяся платформа (ось X или Y)
  logic/
    parseLevel.js           — чистая функция: объект уровня → списки объектов в пикселях
    progress.js             — чистые хелперы: loseLife, isGameOver, nextLevelIndex, isLastLevel
test/
  parseLevel.test.js        — тесты парсера уровней
  progress.test.js          — тесты логики жизней/прогресса
```

## Формат уровней (как добавить свой)

Уровни хранятся в массиве `LEVELS` в `src/levels.js`.

### Карта (`map`)

Массив строк. Каждая строка — один горизонтальный ряд тайлов. Рекомендуемая ширина — 26 символов (832 px при `tileSize = 32`).

| Символ | Объект |
|---|---|
| `=` | статичный тайл (земля/платформа) |
| `^` | шип (хитбокс — нижняя половина тайла) |
| `C` | монета |
| `P` | стартовая позиция игрока (ровно одна) |
| `F` | финишный флаг (ровно один) |
| `.` | пусто |

### Враги (`enemies`)

```js
enemies: [
  { x: 10, y: 6, patrol: [8, 14] }
  // x, y — позиция в тайлах; patrol — [minTileX, maxTileX]
]
```

### Движущиеся платформы (`movingPlatforms`)

```js
movingPlatforms: [
  { x: 11, y: 5, axis: "x", range: 4, speed: 80 }
  // x, y — позиция в тайлах
  // axis  — "x" (горизонтально) или "y" (вертикально)
  // range — дальность движения в тайлах (в каждую сторону от стартовой позиции)
  // speed — скорость в пикселях в секунду
]
```

### Добавление уровня

Добавить новый объект в конец массива `LEVELS`. Размеры мира (`worldWidth`, `worldHeight`) вычисляются автоматически из карты в `parseLevel.js`.

## Чеклист ручного тестирования

Открыть в браузере и пройти все пункты:

- [ ] Меню показывает 3 уровня. Клик / Enter начинает игру.
- [ ] Игрок ходит ←/→.
- [ ] Прыжок ↑/W/Пробел работает только когда стоишь.
- [ ] Игрок стоит на тайлах, не проваливается.
- [ ] Камера следует за игроком.
- [ ] Сбор монеты: монета исчезает, счёт в HUD растёт.
- [ ] Шип → красная вспышка, жизнь −1, уровень перезапускается.
- [ ] Падение в яму → жизнь −1, уровень перезапускается.
- [ ] Прыжок на врага сверху → враг исчезает, игрок отскакивает.
- [ ] Контакт с врагом сбоку → жизнь −1, рестарт.
- [ ] Стоя на движущейся платформе, игрок едет вместе с ней (по X и Y).
- [ ] Можно спрыгнуть и снова запрыгнуть на движущуюся платформу.
- [ ] Касание флага → «УРОВЕНЬ ПРОЙДЕН!». Пробел → следующий уровень с накопленным счётом и свежими 3 жизнями.
- [ ] Потеря всех 3 жизней → «ИГРА ОКОНЧЕНА», R рестартит уровень с 3 жизнями, M — в меню.
- [ ] Прохождение всех 3 уровней → «ИГРА ПРОЙДЕНА!», только M.
- [ ] HUD показывает актуальные жизни/счёт/уровень после смерти и после перехода между уровнями.

## Известные ограничения

- Графика — placeholder (прямоугольники и круги). Инструкция по замене на спрайты — в [ASSETS.md](ASSETS.md).
- Нет звука.
- Прогресс не сохраняется между сессиями браузера.
- Только клавиатурное управление (тач/мобильное не поддерживается).
