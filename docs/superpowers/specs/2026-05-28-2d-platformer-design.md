# 2D Платформер — Дизайн

**Дата:** 2026-05-28
**Статус:** Утверждён к реализации

## Обзор

Веб-игра 2D-платформер про прохождение препятствий. Игрок управляет
персонажем, бегает и прыгает по нескольким вручную спроектированным уровням,
избегает шипов и ям, побеждает врагов прыжком сверху, собирает монеты ради
очков и доходит до флага-финиша.

## Цели и не-цели

**Цели:**
- 3 вручную спроектированных уровня с ощущением прогресса.
- Аркадный платформер-геймплей: ходьба, прыжок, гравитация, столкновения.
- Запуск без сборки: открыть `index.html` в браузере.
- Архитектура, позволяющая позже заменить placeholder-графику на настоящие
  спрайты без переписывания игровой логики.
- Полная документация формата ассетов и уровней.

**Не-цели (YAGNI):**
- Звук и музыка (можно добавить позже).
- Сохранение прогресса между сессиями.
- Мобильное touch-управление (только клавиатура в первой версии).
- Редактор уровней / процедурная генерация.
- Мультиплеер, лидерборды, бэкенд.

## Технологический стек

- **Phaser 3** через CDN (`<script src="...phaser.min.js">`). Без npm, без
  bundler, без build-шага.
- Чистый статический хостинг: `index.html` + папка `src/` с ES-модулями
  (`<script type="module">`).
- Физика: встроенная Arcade Physics Phaser (гравитация, AABB-столкновения).

## Архитектура

### Сцены Phaser

| Сцена | Ответственность |
|-------|-----------------|
| `BootScene` | Генерирует placeholder-текстуры через реестр ассетов; стартует Menu. |
| `MenuScene` | Стартовый экран, заголовок, выбор/старт уровня. |
| `GameScene` | Основной геймплей: строит уровень, спавнит игрока/врагов/монеты/платформы, обрабатывает физику, ввод, столкновения, жизни. |
| `UIScene` | HUD поверх GameScene (запускается параллельно): жизни, счёт монет, номер уровня. Обновляется через события. |
| `LevelCompleteScene` | Экран после касания флага: счёт, кнопка «дальше» (или возврат в меню после последнего уровня). |
| `GameOverScene` | Экран после потери всех жизней: рестарт текущего уровня или выход в меню. |

GameScene и UIScene общаются через Phaser EventEmitter (события
`coin-collected`, `life-lost`, `score-changed`), чтобы UI не лез в игровую
логику напрямую.

### Слой ассетов (`src/assets.js`) — ключевая абстракция

Единый реестр визуалов. Для каждого объекта (`player`, `enemy`, `coin`,
`spike`, `platform`, `tile`, `flag`) задаются: ключ текстуры, размеры, цвет/
форма для placeholder-режима.

В первой версии `BootScene` вызывает `createPlaceholderTextures(scene)` из
этого файла, который рисует каждую текстуру через
`scene.make.graphics().generateTexture(key, w, h)`.

**Переход на настоящие спрайты:** заменяется только реализация в `assets.js`
— вместо `generateTexture` используется `scene.load.image(key, path)` /
`scene.load.spritesheet(...)` плюс описание анимаций. Игровой код ссылается на
объекты только по строковым ключам (`'player'`, `'coin'` и т.д.) и не
изменяется. Контракт (ключи, размеры, ожидаемые анимации) фиксируется в
`ASSETS.md`.

### Формат уровней (`src/levels.js`)

Каждый уровень — обычный JS-объект. Геометрия задаётся сеткой символов
(ASCII-карта), что наглядно и легко редактируется вручную:

```
{
  name: "Уровень 1",
  tileSize: 32,
  map: [
    "..............",
    "..........C...",
    "....===.......",
    "P.......^^...F",
    "==============",
  ],
  legend: {
    "=": "tile",      // твёрдый блок
    "^": "spike",     // шип (смерть при касании)
    "C": "coin",      // монета
    "P": "player",    // точка старта игрока
    "F": "flag",      // финиш
    ".": "empty",
  },
  // динамические объекты, которые сложно выразить в сетке:
  enemies: [{ x, y, patrol: [xMin, xMax] }],
  movingPlatforms: [{ x, y, axis: "x"|"y", range, speed }],
}
```

GameScene парсит `map`, инстанцирует объекты по координатам сетки, затем
добавляет врагов и движущиеся платформы из отдельных массивов. Формат
документируется в `README.md`, чтобы легко добавлять новые уровни.

### Структура файлов

```
index.html
src/
  main.js              # конфиг Phaser, список сцен, запуск игры
  assets.js            # реестр ассетов + генерация placeholder-текстур
  levels.js            # массив из 3 уровней
  scenes/
    BootScene.js
    MenuScene.js
    GameScene.js
    UIScene.js
    LevelCompleteScene.js
    GameOverScene.js
  entities/
    Player.js          # класс игрока: движение, прыжок, состояние
    Enemy.js           # класс врага: патруль, смерть от прыжка сверху
    MovingPlatform.js  # класс движущейся платформы
ASSETS.md              # контракт ассетов для замены на спрайты
README.md              # как запустить, формат уровней, управление
```

## Игровые механики

**Управление:** ←/→ или A/D — движение; ↑ / W / Пробел — прыжок.

**Физика:** Arcade-гравитация тянет игрока вниз. Прыжок только при контакте с
землёй (проверка `body.blocked.down`). Столкновение с твёрдыми тайлами и
платформами останавливает движение.

**Опасности:**
- *Шипы* — касание = потеря жизни и рестарт уровня.
- *Ямы* — падение ниже нижней границы уровня = потеря жизни.

**Враги:**
- Патрулируют в заданном диапазоне по X.
- Касание сбоку/снизу = потеря жизни.
- Прыжок игрока сверху (игрок падает, контакт сверху врага) = враг
  уничтожается, игрок слегка отскакивает.

**Движущиеся платформы:** двигаются по оси X или Y между границами; игрок,
стоящий сверху, везётся вместе с ними.

**Монеты:** при касании исчезают, +очки. Не обязательны для прохождения —
нужны только для счёта.

**Финиш:** касание флага = уровень пройден → LevelCompleteScene.

## Жизни и прогресс

- **3 жизни на уровень.** При входе на уровень счётчик жизней сбрасывается в 3.
- Потеря жизни (шип/яма/враг) → рестарт **текущего** уровня, жизни −1.
- Потеря всех жизней → GameOverScene → рестарт уровня с 3 жизнями или меню.
- Прохождение флага → следующий уровень; после последнего — экран победы и
  возврат в меню.
- Очки (монеты) накапливаются; отображаются в HUD.

## Поток данных

```
main.js (Phaser.Game config)
  └─ BootScene → создаёт текстуры из assets.js → MenuScene
       └─ MenuScene → запускает GameScene(levelIndex) + UIScene
            ├─ GameScene парсит levels[levelIndex], строит мир
            ├─ события → UIScene (жизни/очки)
            ├─ флаг → LevelCompleteScene → GameScene(levelIndex+1)
            └─ жизни=0 → GameOverScene → GameScene(levelIndex) | MenuScene
```

## Обработка ошибок / краевые случаи

- Уровень с некорректным символом в `map` → лог-предупреждение, символ
  трактуется как пустой (игра не падает).
- Отсутствие точки старта `P` → лог-ошибка, дефолтная позиция.
- Падение в яму определяется единым world-bounds-чеком, а не отдельными
  объектами ям.
- Множественные одновременные столкновения (например, шип во время приземления
  на врага) → приоритет у смертельного исхода.

## Тестирование

Поскольку проект без сборки и Phaser завязан на DOM/canvas, основное
тестирование — ручное в браузере по чеклисту в README (прыжок, смерть от шипа,
падение в яму, убийство врага сверху, урон от врага сбоку, сбор монеты, провоз
на платформе, прохождение флага, Game Over, переход между уровнями).

Чистую логику без зависимостей от Phaser (парсер ASCII-карты в список объектов,
расчёт оставшихся жизней) выделяем в отдельные функции и покрываем
лёгкими unit-проверками, запускаемыми в браузере или Node.

## План замены графики (документируется в ASSETS.md)

Для перехода на настоящие спрайты потребуется:
1. Подготовить файлы ассетов (PNG/спрайт-листы) под ключи и размеры из реестра.
2. В `assets.js` заменить генерацию текстур на `load.image`/`load.spritesheet`.
3. Описать анимации (idle/run/jump для игрока, walk для врага) и
   зарегистрировать их в BootScene.
4. Игровой код менять не нужно — он ссылается на текстуры по ключам.
